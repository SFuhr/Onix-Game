# DearTown!
## by Stanislav Khrustalenko
### Специально для Onix-Systems

Немного о проекте: постарался воссоздать максимально точно игровой процесс из видео. Размытую камеру сзади добавил тоже чтобы соответствовало видео, но больше ради прикола. Весь код написан мною, иногда с помощью гугла.

В игре есть проблема, движение влево-вправо зависит фреймрейта. Я честно пытался, но не осилил это пофиксить по-человечески. Проблема в алгоритме движения по пикселям, и слишком поздно я об этом подумал. Но в остальном багов быть не должно. Игра с 60 фпс работает идеально. В конце концов, даже bethesda позволяет себе делать фреймозависимые игры, чем я хуже!? Хех.

Самым сложным было для меня придумать, как сделать динамически изменяющийся меш, который еще и реагирует на объекты. Сделать разрывающуюся бумагу сложно, я даже не представляю как, наверное нужнен какой-нибудь шейдер, или динамически собирать-разбирать меш в коде, но для меня это пока сложно. Поэтому я придумал делать все вычисления в двумерном массиве, и "транслировать" его на сцену. Делать для каждой клеточки GameObject было очевидно не вариант, тогда я научился использовать SetPixels() у текстуры. Сработало, как надо. Остальные фичи сделал без проблем.

Надеюсь, вам понравится!
